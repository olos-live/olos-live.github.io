    API Reference AgentHub Client v0.0.31

    Modules

    AgentHubClient

    This is the main client module. It provides the top-level functions to interact with the AgentHub application, like querying the state of agents, fetching list of agents and sending messages to an agent.

    AgentHubClient.AgentState

    Defines the agent state record to be used when dealing with mnesia and the agent state struct to be used in all the rest of the application.

    AgentHubClient.Cluster

    Provides supporting functions for dealing with node names, like identifying the application in which the client is running based on the node name.

    AgentHubClient.Mnesia.Delete

    Provides functions for deleting data from mnesia.

    AgentHubClient.Mnesia.Init

    This module is responsible for initializing mnesia, subscribing to events and depending on the scenario (standalone or cluster) using the Mnesia.Setup module to create tables or to copy them and joining the cluster.

    AgentHubClient.Mnesia.Insert

    Provides functions for inserting data into the agent's state table in mnesia.

    AgentHubClient.Mnesia.Monitor

    This module listens for events from mnesia like up and down and invoke the corresponding handlers in the Mnesia.Setup module.

    AgentHubClient.Mnesia.Queries

    Provides functions for querying the agents table in mnesia.

    AgentHubClient.Mnesia.Setup

    Provides functions for setting up mnesia in two different scenarios

    AgentHubClient.Mnesia.Updates

    Provides functions that update agent's state record in the mnesia table.

    AgentHubClient.Mnesia.Utils

    Provides common functions used by the other mnesia-modules.

    AgentHubClient.Store

    Provides an interface for interacting with the data store. In this version the data store is mnesia so we delegate all the functions to mnesia-specific modules.

    AgentHubClient.Supervisor

    Supervises mnesia and libcluster processes. The libcluster process is only used if the node is started with a name. Add it to your supervision tree (usually in application.ex). Example
